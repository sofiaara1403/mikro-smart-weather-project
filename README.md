# Uas-smart-weather-mikrokontroler
''
MQTT pada Smart Weather and Distance Monitoring Station

MQTT (Message Queuing Telemetry Transport) merupakan protokol komunikasi yang dirancang khusus untuk sistem Internet of Things (IoT). MQTT digunakan untuk mengirimkan data antar perangkat melalui jaringan internet dengan cara yang ringan, cepat, dan efisien. Dibandingkan dengan protokol komunikasi lainnya, MQTT memiliki ukuran paket data yang kecil sehingga tidak membutuhkan bandwidth yang besar. Oleh karena itu, MQTT sangat cocok diterapkan pada perangkat mikrokontroler seperti ESP32 atau ESP8266 yang memiliki sumber daya terbatas. Dalam proyek Smart Weather and Distance Monitoring Station, MQTT berperan sebagai media komunikasi yang menghubungkan mikrokontroler dengan server atau dashboard monitoring sehingga data hasil pembacaan sensor dapat dikirim dan ditampilkan secara real-time.

Pada sistem ini, mikrokontroler bertugas membaca data dari berbagai sensor, seperti sensor suhu dan kelembapan (DHT22 atau DHT11) serta sensor jarak ultrasonik (HC-SR04). Setelah data diperoleh, mikrokontroler akan mengolahnya terlebih dahulu sebelum mengirimkannya ke MQTT Broker melalui koneksi Wi-Fi. Seluruh proses pengiriman dilakukan secara otomatis tanpa memerlukan permintaan dari aplikasi penerima, sehingga informasi kondisi lingkungan dapat dipantau secara langsung oleh pengguna. Dengan demikian, sistem mampu memberikan informasi cuaca dan jarak secara cepat dan akurat.

Cara kerja MQTT menggunakan konsep Publish-Subscribe, yaitu terdapat tiga komponen utama yang saling berinteraksi. Komponen pertama adalah Publisher, yaitu perangkat yang mengirimkan data. Pada proyek ini, ESP32 berperan sebagai publisher karena bertugas mengirimkan data hasil pembacaan sensor ke broker. Komponen kedua adalah MQTT Broker, yaitu server yang menerima seluruh data dari publisher, kemudian mengatur dan meneruskannya kepada perangkat lain yang membutuhkan informasi tersebut. Broker dapat dianalogikan sebagai kantor pos yang menerima surat dari pengirim, kemudian mendistribusikannya kepada penerima yang sesuai. Komponen ketiga adalah Subscriber, yaitu perangkat atau aplikasi yang menerima data dari broker, seperti dashboard web, aplikasi Android, ataupun Node-RED untuk menampilkan hasil monitoring.

Komunikasi pada MQTT dilakukan menggunakan topic, yaitu sebuah alamat atau jalur yang digunakan sebagai identitas data. Setiap jenis data dapat memiliki topic yang berbeda agar lebih mudah dikelola. Sebagai contoh, data suhu dapat dikirim menggunakan topic weather/temperature, data kelembapan menggunakan weather/humidity, sedangkan data jarak menggunakan weather/distance. Dashboard hanya perlu melakukan proses subscribe pada topic yang diinginkan sehingga setiap ada data baru yang dikirim oleh publisher, broker akan langsung meneruskannya ke dashboard tanpa harus melakukan proses permintaan data secara terus-menerus.

Pada implementasi proyek Smart Weather and Distance Monitoring Station, proses kerja dimulai ketika sensor membaca kondisi lingkungan, seperti suhu udara, kelembapan udara, dan jarak suatu objek. Data tersebut kemudian dikirim ke mikrokontroler ESP32 untuk diproses. Setelah itu, ESP32 akan terhubung ke jaringan Wi-Fi dan melakukan koneksi ke MQTT Broker. Selanjutnya, ESP32 mengirimkan data sensor ke broker melalui topic yang telah ditentukan. Broker kemudian mendistribusikan data tersebut kepada seluruh perangkat yang telah melakukan subscribe, misalnya dashboard monitoring. Dashboard akan menampilkan informasi terbaru secara otomatis sehingga pengguna dapat memantau kondisi lingkungan tanpa perlu melakukan refresh halaman.

Penggunaan MQTT memberikan berbagai keuntungan dalam pengembangan sistem IoT. Salah satu keunggulannya adalah penggunaan bandwidth yang sangat kecil karena hanya data yang diperlukan saja yang dikirimkan. Selain itu, MQTT mampu mengirimkan data dengan cepat sehingga monitoring dapat dilakukan secara real-time. Protokol ini juga mendukung komunikasi antara banyak perangkat sekaligus melalui satu broker, sehingga apabila di kemudian hari sistem ingin dikembangkan dengan menambahkan sensor baru, proses integrasinya menjadi lebih mudah. MQTT juga memiliki tingkat keandalan yang baik karena menyediakan fitur Quality of Service (QoS) yang memastikan data dapat diterima sesuai tingkat jaminan yang dipilih oleh pengembang.

Meskipun memiliki banyak kelebihan, MQTT juga memiliki beberapa keterbatasan. Protokol ini memerlukan koneksi jaringan yang stabil agar proses komunikasi dapat berjalan dengan baik. Selain itu, keberadaan MQTT Broker sangat penting karena seluruh proses komunikasi bergantung pada broker tersebut. Dari sisi keamanan, apabila sistem digunakan pada lingkungan yang lebih luas, diperlukan mekanisme autentikasi, penggunaan username dan password, serta enkripsi menggunakan SSL/TLS agar data yang dikirim tetap aman dari akses pihak yang tidak berwenang.

Berdasarkan penjelasan tersebut, dapat disimpulkan bahwa MQTT merupakan komponen utama dalam proyek Smart Weather and Distance Monitoring Station karena berfungsi sebagai media komunikasi antara mikrokontroler dan dashboard monitoring. Dengan menerapkan mekanisme Publish-Subscribe, MQTT memungkinkan data hasil pembacaan sensor dikirim secara cepat, efisien, dan real-time, sehingga pengguna dapat memantau kondisi cuaca dan jarak secara langsung melalui aplikasi atau website. Penggunaan MQTT juga membuat sistem lebih hemat bandwidth, mudah dikembangkan, serta sangat sesuai untuk implementasi pada berbagai aplikasi berbasis Internet of Things (IoT).
